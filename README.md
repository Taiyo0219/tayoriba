# たよりば

若者が「どこに相談すればよいか分からない」という状態から、困りごとや相談方法に合う相談窓口を探せるWebアプリです。

本アプリは、相談そのものに回答するサービスではなく、相談先を探すための案内サービスとして制作しています。

## 概要

就職、進路、生活費、人間関係、心身の不調などについて悩みがあっても、相談窓口の情報は複数のWebサイトに分散しており、自分がどこへ相談すればよいか判断しにくいことがあります。

たよりばでは、利用者が「困りごと」「地域」「相談方法」などを選択することで、条件に合う相談先を検索できます。何を選べばよいか分からない場合は、探し方ガイドから短い質問に答えて検索条件を整理できます。

## 制作背景

悩みを抱えているときは、相談先を探す作業自体が負担になる場合があります。

また、相談窓口の名称だけを見ても、次のような情報をすぐに判断できないことがあります。

* どのような悩みを相談できるか
* 自分の年齢が対象に含まれるか
* 電話以外の相談方法があるか
* 無料または匿名で相談できるか
* 予約が必要か

そこで、制度名や組織名を知っていることを前提にせず、利用者自身の「困りごと」から相談先を探せるWebアプリを制作しました。

## 主な機能

### 1. トップページの導線整理

トップページでは、最初に「相談先を探す」「探し方を整理する」「相談前に話すことを整理する」の3つから始められるようにしています。

濃い色の大きな訴求や過度なカード表現を避け、相談者が落ち着いて読み進められる画面を目指しました。

### 2. 探し方ガイド

検索条件を自分で選ぶ前に、短い質問から探し方を整理できます。

相談先を断定するのではなく、整理された条件を確認してから検索結果へ進める構成にしています。

<img width="1908" height="946" alt="探し方ガイド画面" src="https://github.com/user-attachments/assets/4370a79b-366b-4008-81f8-aac25001f4a6" />

### 3. 相談先検索

困りごと、相談方法、地域、任意条件を選んで相談先を検索できます。

検索画面には入力状況が分かる進捗表示を追加し、どこまで選べば検索できるかを確認しながら進められるようにしています。

<img width="1906" height="946" alt="進捗表示つき検索画面" src="https://github.com/user-attachments/assets/0475d79b-e205-4b7b-9710-9d354adcbd4b" />

検索条件の例：

* 困りごと
* 地域
* 電話、対面、オンライン、文字相談
* 無料相談
* 匿名相談

### 4. 検索結果の一覧表示

入力した条件に合う相談先を一覧で表示します。

各相談先では、相談先名、概要、相談方法、対応地域、無料相談の可否、匿名相談の可否などを確認できます。

<img width="1913" height="939" alt="検索結果画面" src="https://github.com/user-attachments/assets/9a69aa02-af93-43e7-ba93-73b4d55f5bd3" />

### 5. 相談先の詳細表示

相談先ごとの詳しい情報を確認できます。

掲載情報が変更される可能性を考慮し、利用前に公式サイトを確認するよう案内しています。

<img width="1914" height="944" alt="相談先詳細画面" src="https://github.com/user-attachments/assets/b3197926-dbe5-434a-8bfb-4a59beda7c65" />

詳細画面で確認できる情報：

* 運営組織
* 対象年齢
* 対応する相談内容
* 相談方法
* 電話番号
* 受付時間
* 費用
* 匿名相談の可否
* 予約の要否
* 公式サイト

### 6. お気に入り保存

気になった相談先をお気に入りとして保存できます。

ログイン機能を使わず、ブラウザのlocalStorageに相談先IDを保存しています。

### 7. 相談内容整理メモ

相談する前に、困っていることや相談したい内容を整理できます。

入力内容はlocalStorageに保存されるため、同じブラウザであればページを閉じた後も確認できます。個人情報をサーバーへ送信しない構成にしています。

## UI/UXで意識した点

相談先を検索するアプリでは、効率だけでなく、利用者が検索前に感じる不安を減らすことも重要だと考えました。

そのため、以下の点を意識してUIを改善しています。

* 入力している段階が分かる進捗表示
* 「まだ決められない」を許容する選択肢
* 相談者を急かさない、落ち着いた画面設計
* 相談方法や地域を無理に決めさせない導線
* 検索前に探し方を整理できるガイドページ
* キーボード操作時のfocus-visible表示
* スマートフォンでも読みやすい1カラム表示

## システム構成

```text
ブラウザ
  ↓ Fetch API
Node.js / Express
  ↓ MongoDB Driver
MongoDB Atlas
```

相談先情報はMongoDB Atlasに保存し、Node.js・Expressで作成したAPIから取得します。

お気に入りと相談メモは、個人情報をサーバー側へ保存しないため、ブラウザのlocalStorageに保存しています。

## セットアップ方法

### 1. リポジトリを取得

```bash
git clone リポジトリのURL
cd tayoriba
```

### 2. パッケージをインストール

```bash
npm install
```

### 3. 環境変数を設定

`.env.example`をコピーして`.env`を作成します。

```bash
cp .env.example .env
```

Windows PowerShellの場合：

```powershell
Copy-Item .env.example .env
```

`.env`にMongoDB Atlasの接続文字列を設定します。

```env
MONGODB_URI=MongoDB Atlasの接続文字列
PORT=3000
```

### 4. 初期データを投入

```bash
node seed.js
```

### 5. アプリを起動

```bash
npm run dev
```

ブラウザで以下を開きます。

```text
http://localhost:3000
```


## 実装上の工夫

### 利用者の困りごとを起点にした検索

相談機関名や制度名を知っていることを前提にせず、利用者が自分の悩みに近いカテゴリを選ぶだけで検索できる構成にしました。

### 個人情報を保存しない設計

MVPでは会員登録を実装せず、お気に入りや相談メモはlocalStorageに保存しています。

これにより、利用者の相談内容をサーバー側へ保存しない構成にしています。

### 画面表示とAPIの役割分担

画面表示はHTML・CSS・JavaScript、相談先の取得はNode.js・Express、データ保存はMongoDB Atlasと役割を分けました。

JavaScript共通関数を分け、検索結果、詳細、お気に入り、メモ画面で必要な処理を再利用しやすくしています。

## 注意事項

本アプリは、ポートフォリオ用に制作したMVPです。

現在登録されている相談先情報には、機能確認を目的とした架空データが含まれています。表示される電話番号やWebサイトは、実際の相談には利用できません。

本アプリは、医療行為、法律相談および緊急対応を行うものではありません。

今後、実在する相談窓口を掲載する場合は、各機関の公式情報を確認し、情報の出典と最終確認日を明記する予定です。
